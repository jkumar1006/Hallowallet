### Technical Questions

**Q: "How does the NLP parsing work?"**
A: "We built a custom pattern-matching engine that uses regex and contextual analysis. It identifies amounts, categories, dates, and descriptions from natural language. No external APIs - all processing happens client-side for privacy and speed."

**Q: "What's the performance impact of real-time analysis?"**
A: "Minimal. We use parallel API fetching with Promise.all(), React optimization, and smart caching. Initial load is under 500ms, and background updates don't block the UI."

**Q: "How scalable is this architecture?"**
A: "Very scalable. We use Next.js API routes, JSON file storage (easily migrated to PostgreSQL), and stateless authentication with JWT. The system can handle thousands of concurrent users."

**Q: "Why not use a real ML model?"**
A: "For a hackathon, we focused on rule-based AI that delivers immediate value. The architecture is designed to easily integrate ML models for predictive insights in the future."

A: "Freemium model - basic features free, premium features ($5/month) include advanced insights, unlimited history, and bank integration. Affiliate revenue from retailer links."

**Q: "Who's the target market?"**
A: "Young professionals (22-35) in India who want to improve their financial habits but find traditional apps too complex or boring."

**Q: "What's the competitive advantage?"**
A: "Proactive AI that delivers value without user effort. Most apps require manual work - we automate the analysis and surface insights automatically."

**Q: "How do you acquire users?"**
A: "Content marketing (financial tips), social media (Instagram/Twitter), app stores (ASO), and partnerships with Indian fintech companies."
